## Overview

The `CasosController` manages legal cases in the Sistema de Abogados application. It handles case creation, updates, deletion, search functionality, and relationship management with processes and opposing parties.

**Namespace:** `App\Http\Controllers`

**Extends:** `Controller`

## Dependencies

The controller uses the following models:
- `Casos` - Main case model
- `Cliente` - Client model
- `TipoProceso` - Process type model
- `ParteContraria` - Opposing party model
- `Caso_has_proceso` - Case-Process pivot model
- `Caso_has_PContraria` - Case-Opposing party pivot model
- `CasoDocumento` - Case document model

## Methods

### index()

Lists all cases with pagination and search functionality.

**Purpose:** Display a paginated list of cases with optional search filtering

**Parameters:** None (uses `request('search')` for search query)

**Returns:** View `caso.caso.index` with cases and clients

**Response Data:**
<ResponseField name="casos" type="Collection">
  Paginated collection of cases (5 per page)
</ResponseField>
<ResponseField name="clientes" type="Collection">
  All clients for filtering/assignment
</ResponseField>

**Code Example:**
```php
public function index()
{
    $casos = Casos::search(request('search'))->paginate(5);
    $clientes = Cliente::all();
    return view('caso.caso.index', compact('casos','clientes'));
}
```

**Route:** `caso.caso.index`

### store()

Creates a new case with validation.

**Purpose:** Store a new case in the database

**Request Parameters:**
<ParamField path="n_caso" type="string" required>
  Case number or identifier
</ParamField>
<ParamField path="fecha_inicio" type="date" required>
  Case start date
</ParamField>
<ParamField path="fecha_final" type="date">
  Case end date (optional)
</ParamField>
<ParamField path="descripcion" type="text">
  Case description (optional)
</ParamField>
<ParamField path="estado" type="string" required>
  Case status
</ParamField>
<ParamField path="id_cliente" type="integer" required>
  Client ID associated with the case
</ParamField>

**Validation Rules:**
```php
[
    'n_caso' => 'required', 
    'fecha_inicio' => 'required',
    'fecha_final' => 'nullable',
    'descripcion' => 'nullable',
    'estado' => 'required',
    'id_cliente' => 'required'
]
```

**Returns:** Redirect to `caso.caso.index` with success message

**Success Message:** "Caso registrado correctamente."

**Code Example:**
```php
public function store(Request $request)
{
    $validated = $request->validate([
        'n_caso' => 'required', 
        'fecha_inicio' => 'required',
        'fecha_final' => 'nullable',
        'descripcion' => 'nullable',
        'estado' => 'required',
        'id_cliente' => 'required'
    ]);
    Casos::create($validated);
    return to_route('caso.caso.index')
        ->with('message', 'Caso registrado correctamente.');
}
```

**Route:** `caso.caso.store`

### destroy()

Deletes a case from the database.

**Purpose:** Remove a case with foreign key constraint handling

**Parameters:**
<ParamField path="caso" type="Casos" required>
  Case model instance (route model binding)
</ParamField>

**Returns:** 
- Success: Redirect to `caso.caso.index` with success message
- Error: Back with alert message if constraints exist

**Success Message:** "Caso eliminado correctamente."

**Error Message:** "No puede eliminar! Antes elimine sus conexiones."

**Code Example:**
```php
public function destroy(Casos $caso)
{
    try{
        $caso->delete();
        return to_route('caso.caso.index')
            ->with('messagedestroy', 'Caso eliminado correctamente.');
    }catch(\Exception $e){
        return back()->with('messageAlert', 
            'No puede eliminar! Antes elimine sus conexiones.');
    }
}
```

**Route:** `caso.caso.destroy`

**Notes:** 
- Uses try-catch to handle foreign key constraint violations
- Related entities must be removed before deleting the case

### assignProceso()

Assigns a process type to a case.

**Purpose:** Create a relationship between a case and a process type

  Case model instance (route model binding)
</ParamField>

**Request Parameters:**
<ParamField path="id_tipo_proceso" type="integer" required>
  Process type ID to assign
</ParamField>

**Returns:** Back to previous page with success message

**Success Message:** "Proceso asignado."

**Code Example:**
```php
public function assignProceso(Request $request, Casos $caso)
{
    $validated = $request->input();
    $validated['id_caso'] = $caso->id;
    Caso_has_proceso::create($validated);
    return back()->with('message', 'Proceso asignado.');
}
```

```php

$data = [
    'n_caso' => 'CASO-2026-001',
    'fecha_inicio' => '2026-03-10',

    'estado' => 'Activo',
    'id_cliente' => 5
```

```php
// PUT /casos/{caso}
$data = [
    'n_caso' => 'CASO-2026-001',
```

## Error Handling

### Deletion Constraints

When attempting to delete a case with related records:

```php
// If foreign key constraints exist

    'No puede eliminar! Antes elimine sus conexiones.');
```

### Validation Errors

Laravel's validation will automatically return error messages for:
- Missing required fields
- Invalid data formats
- Type mismatches
